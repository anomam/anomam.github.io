Setting up my Python environment: switching between 2.7, 3.6, and virtualenvs

Posted on Sun 08 April 2018 in Python

I am really excited to share this blog post from Henrique Bastos on using pyenv and virtualenvwrapper in order to set up an amazing and flexible Python environment.

This is exactly what I was looking for in order to decrease my reliance on 2.7 and to finally switch to Python 3!

I encountered only one little issue when I tried to import matplotlib in a notebook in Jupyter, within a Python 2 virtual environment. And I was able to solve it with the following steps:

  • I installed ipykernel in my Python 2 virtual environment using the usual method:
pip install ipykernel
  • I then installed the kernel for jupyter while giving it a name that would help me recognize it, like py2_my_environment:
python -m ipykernel install --user --name py2_my_environment

After opening Jupyter, and then starting a new notebook using the Kernel named py2_my_environment, I am now able to import matplotlib without any problem!

I hope this is helpful to others that followed the same post to set up their Python environment :)
